# Termlnk 文档

Termlnk 是一个面向开发者的智能终端，把本地终端、SSH/SFTP、端口转发、代码片段、AI Agent、MCP 与插件系统放在同一个工作区里。

## 功能地图

| 功能 | 适合场景 |
| --- | --- |
| 终端工作区 | 本地 PTY、SSH 会话、递归分屏、拖拽调整大小和会话恢复 |
| SSH 连接 | 管理主机树、认证方式、ProxyJump、SOCKS5 代理和 X11 转发 |
| SFTP 文件管理 | 在本地和远程双栏之间传输、预览、重命名和编辑权限 |
| 端口转发 | 通过 SSH 建立 Local、Remote、Dynamic 三类转发规则 |
| 代码片段 | 保存常用脚本，按分组管理，并对指定 SSH 主机执行 |
| AI Agent | 在终端旁进行多轮对话，调用 MCP 工具，并通过审批执行命令 |
| 插件系统 | 通过命令、菜单、UI 部件、设置和主题贡献点扩展应用 |

## 推荐阅读顺序

1. 先阅读 [快速上手](./quick-start)，完成安装、创建终端和连接 SSH 主机。
2. 阅读 [终端工作区](./terminal) 和 [SSH 连接](./ssh)，了解日常使用的基础模型。
3. 如果需要访问内网服务，阅读 [端口转发](./port-forwarding)。
4. 如果有重复运维脚本，阅读 [代码片段](./snippets)。
5. 需要 AI 或自动化工具时，再阅读 [AI Agent](./ai-agent) 与 [MCP 工具](./mcp)。

## 本地优先

Termlnk 默认把会话、主机配置、代码片段、端口转发规则和工作区状态保存在本地。AI 请求只会发送到你配置的 Provider；Web 版则运行在你自托管的服务器上。
