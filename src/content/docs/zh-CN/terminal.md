# 终端工作区

Termlnk 的终端工作区支持本地与远程会话并排排列，以递归分屏布局组织多个面板。

## 本地 PTY 会话

每个终端面板运行一个独立的本地伪终端（PTY）。支持你系统上已安装的任意 Shell（bash、zsh、fish、PowerShell 等）。

## 递归分屏

工作区采用二叉树布局模型，可无限嵌套拆分：

- **水平拆分** — 左右并排
- **垂直拆分** — 上下排列
- 支持任意层级的递归嵌套

## 拖拽调整大小

拖拽面板之间的分隔线即可调整各面板占比，布局状态会自动保存。

## 聚焦放大（Magnify）

在多面板工作区中，可将某个面板临时放大至全屏，专注于当前任务。再次触发即恢复原始布局。

## Shell 集成

Termlnk 实现了 OSC 633 协议，提供：

- 命令块追踪（识别每条命令的输入/输出边界）
- 工作目录同步
- 终端通知
- AI Agent 的 `terminal_run` 工具集成
- 代码片段向指定终端会话粘贴或执行命令

## 代码片段

常用命令可以保存为 [代码片段](./snippets)。片段既可以发送到当前终端会话，也可以绑定多个 SSH 主机并批量执行。

## 终端渲染

基于 xterm.js，支持以下渲染特性：

| 特性 | 说明 |
| --- | --- |
| True Color | 24-bit 真彩色 |
| 连字（Ligatures） | 支持编程字体连字渲染 |
| Sixel | 终端内显示图片 |
| 图像协议 | iTerm2 / Kitty 图像协议 |
| 缓冲区搜索 | 在终端输出中查找文本 |
| 超链接 | 自动检测并可点击 URL |
| IME 输入 | 完整的输入法编辑器支持 |

## 会话恢复

重启应用后，Termlnk 会自动恢复：

- 工作区布局（分屏结构与面板大小）
- 终端会话（本地和远程）
- 滚动缓冲区内容
