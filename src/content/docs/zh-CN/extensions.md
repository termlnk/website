# 插件系统

Termlnk 采用模块化插件架构，允许通过插件扩展应用的功能。

## 架构概述

Termlnk 的插件系统借鉴 VS Code 的扩展模型，基于以下核心概念：

- **贡献点（Contribution Points）** — 插件向应用注册功能的标准化接口
- **隔离运行时** — 插件在独立的运行环境中执行
- **TypeScript API** — 稳定的 API 供插件调用

## 贡献点

插件可以通过以下贡献点向应用注册功能：

| 贡献点 | 说明 |
| --- | --- |
| Commands | 可由 UI、快捷键或其他插件调用的操作 |
| Menus | 将命令注册到菜单（右键菜单、下拉菜单等） |
| UI Parts | 向内置 UI 位置注入 React 组件 |
| Settings | 用户可配置的插件设置项 |
| Keybindings | 键盘快捷键绑定 |
| Themes | 视觉主题 |

## 插件市场

内置插件市场 UI，支持：

- 浏览和搜索可用插件
- 一键安装
- 启用 / 禁用已安装的插件

> **注意：** 插件市场处于早期阶段，目录仍在持续扩充中。

## 插件清单（Manifest）

每个插件需要声明一个清单文件，描述：

- 插件基本信息（名称、版本、描述）
- 贡献点声明
- 依赖关系
- 激活条件

## TypeScript API

Termlnk 提供稳定的 TypeScript API，插件可通过它：

- 注册和执行命令
- 读写设置
- 与 UI 交互
- 访问终端、SSH、AI 等服务

## React 组件注入

插件可以向应用的内置 UI 位置注入自定义 React 组件，实现深度界面集成。
