# 快捷键

Termlnk 支持完整的键盘驱动工作流，提供可自定义的快捷键系统。

## 默认快捷键

以下为常用操作的默认快捷键（macOS / Windows & Linux）：

| 操作 | macOS | Windows / Linux |
| --- | --- | --- |
| 新建终端 | `⌘ N` | `Ctrl+N` |
| 关闭面板 | `⌘ W` | `Ctrl+W` |
| 水平拆分 | `⌘ D` | `Ctrl+D` |
| 垂直拆分 | `⌘ ⇧ D` | `Ctrl+Shift+D` |
| 聚焦放大 | `⌘ ⇧ M` | `Ctrl+Shift+M` |
| 切换面板 | `⌘ [` / `⌘ ]` | `Ctrl+[` / `Ctrl+]` |
| 缓冲区搜索 | `⌘ F` | `Ctrl+F` |
| 命令面板 | `⌘ ⇧ P` | `Ctrl+Shift+P` |
| 设置 | `⌘ ,` | `Ctrl+,` |

> **提示：** 以上为示例快捷键，实际默认值请以应用内设置为准。

## 自定义快捷键

在设置 → 快捷键中，可以：

- 修改任意操作的快捷键绑定
- 为未绑定的命令添加快捷键
- 重置为默认值

## 快捷键上下文

快捷键可以限定在特定上下文中生效：

- **全局** — 在应用任何位置生效
- **终端聚焦时** — 仅当终端面板获得焦点时生效
- **编辑器聚焦时** — 仅在输入框或编辑器中生效

上下文机制避免了不同功能区域之间的快捷键冲突。

## 冲突处理

当多个命令绑定了相同的快捷键时，Termlnk 会：

1. 按上下文优先级匹配（更具体的上下文优先）
2. 在设置中高亮显示冲突的绑定
3. 允许用户手动解决冲突
