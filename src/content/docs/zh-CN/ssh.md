# SSH 连接

Termlnk 内置完整的 SSH 客户端，与终端工作区深度集成。

## 主机树管理

- **层级分组** — 按项目、环境或团队组织主机
- **标签** — 为主机添加标签，快速筛选
- **拖拽排序** — 拖拽主机或分组调整顺序

## 认证方式

支持三种认证方式：

| 方式 | 说明 |
| --- | --- |
| 密码 | 直接输入密码 |
| 私钥 | 指定本地私钥文件路径，支持 passphrase |
| SSH Agent | 使用系统 SSH Agent 中已加载的密钥 |

## ProxyJump 链

通过 `ProxyJump` 配置多跳连接，适用于跳板机场景：

```
本地 → 跳板机 A → 跳板机 B → 目标服务器
```

在主机配置中添加一个或多个跳板机即可，Termlnk 自动建立链式连接。

## SOCKS5 代理

为 SSH 连接配置 SOCKS5 代理，适用于需要通过代理服务器访问的网络环境。

## X11 转发

启用 X11 Forwarding 后，可在本地显示远程服务器上的图形界面程序。

## 端口转发

支持三种端口转发模式，详见 [端口转发](./port-forwarding)：

| 类型 | 参数 | 用途 |
| --- | --- | --- |
| 本地转发（Local） | `-L` | 将远程端口映射到本地 |
| 远程转发（Remote） | `-R` | 将本地端口暴露给远程 |
| 动态转发（Dynamic） | `-D` | 建立本地 SOCKS 代理 |

端口转发规则通过 SSH 主机 ID 引用主机配置，因此会复用主机的认证方式、ProxyJump 链、SOCKS5 代理和主机密钥验证。

## 启动代码片段

SSH 主机可以绑定一个启动代码片段。新建 SSH 会话时，Termlnk 会解析该片段内容并作为启动脚本运行。适合自动进入项目目录、附加 tmux 会话或执行固定初始化命令。详见 [代码片段](./snippets)。

## 主机密钥验证

采用 TOFU（Trust On First Use）策略：首次连接时保存主机指纹，后续连接自动验证。指纹变更时会发出警告。

## 与终端工作区集成

SSH 会话以普通面板的形式出现在工作区中，可以：

- 与本地终端并排分屏
- 参与递归分屏布局
- 支持 Magnify 放大
- 随工作区一起恢复
