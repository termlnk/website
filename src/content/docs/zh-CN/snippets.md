# 代码片段

代码片段用于保存常用命令或脚本，并按分组组织。一个片段可以绑定多个 SSH 目标主机，执行时 Termlnk 会为每个目标创建 SSH 会话并运行脚本。

![代码片段树](/images/docs/termlnk-snippets-tree.png)

## 片段树

代码片段面板位于侧边栏的 `{}` 入口。顶部两个按钮分别用于新建分组和新建片段。

片段树支持：

- 按分组组织片段。
- 展开 / 收起分组，并记住展开状态。
- 拖拽片段或分组调整位置。
- 右键片段执行、编辑、复制或删除。
- 右键分组重命名或删除。

## 创建或编辑片段

片段编辑窗口包含名称、描述、分组、脚本和执行目标。

![代码片段编辑](/images/docs/termlnk-snippet-edit-form.png)

字段说明：

| 字段 | 说明 |
| --- | --- |
| 名称 | 片段在树中的显示名称，必填 |
| 描述 | 片段用途说明，可选 |
| 分组 | 片段所属分组，未选择时放在根目录 |
| 脚本 | 要粘贴或执行的命令内容，最多 65536 个字符 |
| 执行目标 | 从 SSH 主机树中选择一个或多个目标主机 |

## 执行与粘贴

片段右键菜单包含「执行」和「粘贴」相关操作。

- **执行** — 只在片段已配置执行目标时可用。Termlnk 会依次为每个目标主机创建 SSH 会话，等待会话 ready 后运行脚本。
- **粘贴 / 运行到当前会话** — 底层能力由 `snippet.paste(sessionId, content)` 与 `snippet.run(sessionId, content)` 提供，用于把片段内容发送到指定终端会话。

执行多个目标时，请确保脚本本身具备幂等性，或者在脚本中显式处理重复执行、失败重试和权限提升。

## SSH 启动片段

SSH 主机配置支持 `startupSnippetId`。当主机绑定启动片段后，新建 SSH 会话时会读取该片段内容作为启动脚本执行。

这个机制适合：

- 登录后自动切换目录。
- 自动进入 tmux / screen 会话。
- 初始化远程环境变量。
- 执行固定的诊断命令。

如果主机同时配置了旧的 inline 启动脚本和 `startupSnippetId`，启动片段会优先生效。

## 与云同步的关系

代码片段的数据模型包含名称、内容、描述、分组、目标主机 ID、排序和收藏状态。它们可以与主机、设置等本地数据一起参与同步；同步后，片段树会响应远端变更并刷新。

## 使用建议

- 对生产环境脚本使用清晰名称，例如 `restart api service` 或 `tail nginx errors`。
- 把危险操作拆成可审阅的小脚本，避免在一个片段中混合部署、删除和重启。
- 为需要目标主机的片段显式配置执行目标，减少误运行到错误会话的概率。
- 对会改动系统状态的脚本保留注释，说明前置条件和预期结果。
